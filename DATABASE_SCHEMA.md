# Database Schema (Version 2)
## SymptoMap: PostgreSQL + PostGIS Architecture

This schema represents the migration from V1 (SQLite) to an enterprise-grade V2 architecture utilizing PostgreSQL and the PostGIS extension for advanced geospatial capabilities.

---

## 1. PostGIS Enablement
```sql
-- Enable PostGIS extension for geospatial data types and indexing
CREATE EXTENSION IF NOT EXISTS postgis;
-- Enable UUIDossp for secure, decentralized ID generation
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

## 2. Core Operational Tables

### Table: `users`
*(Replacing the single shared password)*
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL CHECK (role IN ('ADMIN', 'EPIDEMIOLOGIST', 'OFFICER', 'REPORTER', 'VIEWER')),
    jurisdiction_polygon GEOMETRY(Polygon, 4326), -- For Row-Level Security masking
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP WITH TIME ZONE
);
```

### Table: `cases`
*(Replacing `doctor_outbreaks` - Optimized for high throughput)*
```sql
CREATE TABLE cases (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    disease_type VARCHAR(100) NOT NULL,
    patient_count INTEGER NOT NULL CHECK (patient_count > 0),
    severity VARCHAR(20) NOT NULL,
    -- PostGIS Geometry Point (Longitude, Latitude) Spatial Reference 4326 (WGS 84)
    location GEOMETRY(Point, 4326) NOT NULL,
    facility_id UUID REFERENCES facilities(id), -- Nullable if community reported
    symptoms JSONB, -- Unstructured data for ML parsing
    reported_by UUID REFERENCES users(id),
    date_reported TIMESTAMP WITH TIME ZONE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    status VARCHAR(20) DEFAULT 'ACTIVE'
);

-- Crucial GiST Index for Lightning Fast Geospatial Queries
CREATE INDEX idx_cases_location ON cases USING GIST (location);
-- B-Tree index for time-series filtering
CREATE INDEX idx_cases_date_reported ON cases (date_reported DESC);
-- Index for JSONB payload querying
CREATE INDEX idx_cases_symptoms ON cases USING GIN (symptoms);
```

### Table: `facilities`
*(New in V2: Managing reporting hospitals/clinics)*
```sql
CREATE TABLE facilities (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(255) NOT NULL,
    facility_type VARCHAR(100),
    capacity_beds INTEGER,
    location GEOMETRY(Point, 4326) NOT NULL,
    region VARCHAR(100)
);
CREATE INDEX idx_facilities_location ON facilities USING GIST (location);
```

## 3. Intelligence & ML Tables

### Table: `clusters`
*(Generated asynchronously by the ML microservice utilizing DBSCAN)*
```sql
CREATE TABLE clusters (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    disease_type VARCHAR(100) NOT NULL,
    bounding_polygon GEOMETRY(Polygon, 4326) NOT NULL,
    total_cases INTEGER,
    risk_level VARCHAR(20),
    ai_summary TEXT, -- Generated by the LLM Agent
    active_from TIMESTAMP WITH TIME ZONE,
    resolved_at TIMESTAMP WITH TIME ZONE,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_clusters_polygon ON clusters USING GIST (bounding_polygon);
```

### Table: `forecast_runs`
*(Stores the outputs of the SEIR and Prophet models)*
```sql
CREATE TABLE forecast_runs (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    disease_type VARCHAR(100) NOT NULL,
    target_region GEOMETRY(Polygon, 4326) NOT NULL,
    model_type VARCHAR(50) NOT NULL, -- e.g., 'SEIR', 'PROPHET'
    parameters JSONB, -- e.g., {"R0": 1.5, "incubation": 3}
    trajectory_data JSONB NOT NULL, -- The time-series array of predicted cases
    confidence_interval DECIMAL(4,2),
    executed_by UUID REFERENCES users(id),
    executed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## 4. Alerting System

### Table: `alert_rules`
*(User-defined triggers)*
```sql
CREATE TABLE alert_rules (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID REFERENCES users(id),
    disease_type VARCHAR(100),
    threshold_count INTEGER NOT NULL,
    time_window_hours INTEGER NOT NULL,
    monitored_polygon GEOMETRY(Polygon, 4326) NOT NULL, -- Trigger if cases in this area exceed threshold
    notification_channels JSONB -- e.g., ["EMAIL", "SMS"]
);
```
